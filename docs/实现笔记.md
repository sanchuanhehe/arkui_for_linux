## 1. 渲染呈现管线(C2-b,已截图验证)

**没走 Android 直驱,走软光栅纹理上传**(blast radius 最小,复用全部已工作部分)。

链路:
```
RS 线程: RSSurfaceWindows::FlushFrame (graphic_2d .../rs_surface_windows.cpp:89-132)
          把页面树渲成 RGBA8888 CPU 位图 → 调 onRender_(addr,size,w,h,ts)
       → WindowView::OnRsFrame (static OnRenderFunc)  [mutex 拷进 latestFrame_]
主线程: WindowView::Present  [惰性建 GLES2 program/texture/quad,
          上传 latestFrame_ → 画全屏纹理 quad → wl_egl_window EGLSurface → swap]
```

- linux RS 走 **windows 平台组**(`render_service_base/src/platform/BUILD.gn:20`
  `rosen_is_win||current_os=="linux"`→`windows`),`RSSurfaceWindows` 经 `OnRenderFunc`
  纯函数指针回调把位图交出来——这是不碰 rosen/平台组就能上屏的关键缝。
- **旧 bug(必记)**:`CreateSurfaceNode` 当时把 `eglWindow_` 当 `additionalData` 传进去
  → 被 cast 成 `OnRenderFunc` 调用 `wl_egl_window*` → 崩。因为没 `.ets` 一直没触发,潜伏很久。
  **修法**:`CreateSurfaceNode` 传 `&WindowView::OnRsFrame`(真 `OnRenderFunc`)。
- **UV 翻 Y**:RS 位图是 top-down,GL 纹理坐标要翻 Y 才正立。
- **单窗口**:静态 `instance_` 指针(OnRenderFunc 是无 this 的函数指针,需静态桥)。
- **截图**:`ACE_SHOT_PPM=<path>` env → `Present` 里 `glReadPixels`→PPM(Y 翻转)。
  weston-screenshooter 在 headless 断言 `width>0` 不可用,所以走客户端侧抓帧。
- **自测帧**:`ACE_TEST_FRAME=1` 注入青/品红/黄三条带,**走 OnRsFrame 同一条路**——
  验证呈现管线时不依赖 RS 真页面树。截图实测三条带正立、像素精确。

相关文件:`entrance/WindowView.{h,cpp}`(`Present` ~730-784、`OnRsFrame`)、
`entrance/virtual_rs_window.cpp`(`CreateSurfaceNode` ~797,RS 驱动块 gate
`#if MAC_PLATFORM||LINUX_PLATFORM`)。

## 2. 事件驱动主循环(架构要点:事件订阅,非忙等)

`main.cpp` **弃** `while(ctx.Dispatch())` 忙等,改 `WaylandContext::RunEventLoop()`:

- 把 wayland fd(`wl_display_get_fd`)经 `EventHandler::AddFileDescriptorListener(fd,
  FILE_DESCRIPTOR_INPUT_EVENT)` **订阅进主线程 ability EventRunner 的 EpollIoWaiter**
  (底层 `epoll_wait`)。
- 单个阻塞 `for (auto e = queue->GetEvent(); e; ...)` 循环**同时 pump**:ability 任务 +
  wayland 事件(`WaylandFdListener::OnReadable`→`wl_display_dispatch`+`flush`;
  frame callback→`Present`)。零 spin / 零 sleep。
- **根因(为什么必须这样)**:`AppMain::DispatchOnCreate` 用 `eventHandler_->PostTask`
  投到 `EventRunner::Current()`(主线程);旧主循环只 `wl_display_dispatch`、不 pump 该队列
  → **ability 任务永不执行**(OnRsFrame 永远 0)。整合后 ability 生命周期端到端跑通:
  `DispatchOnCreate→LoadModule→FindAbility→OnWindowStageCreated→OnForeground`。

## 3. Stage 启动接线

`StageViewController::LoadView()`(镜像 mac `viewDidLoad`),**顺序关键**:

```
new WindowView → NotifySurfaceChangedWithWidth(480,800,1.0) → AddWindowView
  → ShowOnView → CreateSurfaceNode → StartBaseDisplayLink → AppMain DispatchOnCreate/Foreground
```

`ShowOnView`/`CreateSurfaceNode` **必须在** `StartBaseDisplayLink` **前**,否则 `Present`
还没 `wlSurface_`、arm 不上 frame callback。

`MacAppDelegate::ApplicationDidFinishLaunching`(`entrance/MacAppDelegate.cpp:42`)用 env
`ACE_STAGE_LAUNCH` gate:有 bundle 走 stage 启动(`ConfigModuleWithBundleDirectory("arkui-x")`
+`LaunchApplication`+`new StageViewController(...)->LoadView()`),否则裸窗口(不破坏 Stage C)。

## 5. 平台事实(避坑)

- **链 GCC-13 libstdc++(非 libc++)**:OHOS linux-aarch64 实际链 libstdc++;libc++ 内部用法
  (`std::optional::__get()` 等)会炸,缺标准头(`<memory>/<cstdint>/<mutex>`)要补。
- **本机即 aarch64**,直接跑 `ace_linux`,无需模拟器。软渲染 llvmpipe(`LIBGL_ALWAYS_SOFTWARE=1
  GALLIUM_DRIVER=llvmpipe`)。
- 编/跑/截图标准流程见 `.claude/skills/build-run-verify/SKILL.md`。

## 6. 已弃/待续的路

- **(弃)Android 直驱 `RSSurfaceLinux`**:理论更优(Wayland 无 mac 的 CAOpenGLLayer
  独占约束),但切平台组牵动 windows 专属源,链接缺口爆发。C2-b 软光栅上传已够用。
- **(待)C2-c `ROSEN_ARKUI_X`**:`config.gni:82` 加该宏激活客户端 GPU 收尾,全树爆炸半径最大,
  需增量(局部 `-DROSEN_ARKUI_X` 验证再推全局)。当前不需要。
- **Stage D 真 .ets 上屏**:呈现管线 + ability 生命周期都已通,**只差结构正确的 bundle**。
  根因(pkgContextInfo→IsNormalizedOhmUrlPack→record 解析)与官方 SDK 续接路见
  journal 文档。
